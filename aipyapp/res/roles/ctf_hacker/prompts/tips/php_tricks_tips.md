# PHP 特性与黑魔法 (PHP Tricks) 漏洞指南

PHP 题目在 CTF Web 中占据半壁江山。这类题目通常不涉及大型框架，而是考验你对 PHP 语言底层特性、弱类型比较、正则表达式陷阱以及内置函数的深入理解。

## 1. 弱类型比较 (Loose Comparison)

PHP 中 `==`（弱比较）会在比较前进行类型转换，常常导致逻辑绕过。

- **字符串与数字比较**:
  - `"0e12345" == "0e54321"` -> `true` (科学计数法，0的任何次方都是0)
  - `"1admin" == 1` -> `true` (PHP 8 之前，字符串会被转换为开头的数字)
  - `md5("240610708") == md5("QNKCDZO")` -> `true` (两者 MD5 值都是 `0e` 开头)
- **绕过技巧**:
  - 寻找具有 `0e` 开头 Hash 值的魔法字符串字典（网上一搜一大把）。
  - 如果比较的是数组：`[1,2,3] == [1,2,3]`。有时传入数组 `?a[]=1&b[]=2` 可以让某些期望接受字符串的函数返回 `NULL`，进而导致 `NULL == NULL` 成立。

## 2. 强类型绕过 (Strict Comparison)

即便使用了 `===`，也可以利用特定函数的缺陷绕过。

- **MD5/SHA1 数组绕过**:
  - `md5($_GET['a']) === md5($_GET['b'])` 且 `$a !== $b`。
  - 解法：传入数组 `?a[]=1&b[]=2`。`md5()` 处理数组会返回 `NULL`，发出 Warning，但流程继续，最终 `NULL === NULL` 成立。
- **MD5 碰撞 (Hash Collision)**:
  - 提交两个内容不同但 MD5 原像完全相同的 PDF 或二进制数据（需要编写 Python 脚本发送二进制 Payload）。

## 3. 函数解析特性与参数名绕过

- **变量名解析 (Variable Parsing)**:
  - 当 PHP 将查询字符串（如 `?a.b=1`）解析为 `$_GET` 变量时，会将特定的字符（如空格 ` `, 点号 `.`, 左方括号 `[`）转换为下划线 `_`。
  - 绕过：如果 WAF 过滤了 `_`，但后端又需要 `my_var`，你可以传递 `?my.var=1` 或 `?my var=1`。
- **`extract()` 和 `parse_str()`**:
  - 这两个函数会将数组中的键值对注册为变量，极易造成**变量覆盖 (Variable Overriding)**。
  - 如 `extract($_GET);` 允许你覆盖后续代码中的任何核心变量。

## 4. 正则表达式绕过 (Regex Bypass)

- **`preg_match()` 绕过**:
  - 数组绕过：传入数组 `?a[]=` 可以让 `preg_match` 返回 `false`。
  - 回溯次数限制 (Backtrack Limit)：PHP 的 PCRE 回溯次数默认限制为 100 万。如果传递一个超长的无效字符串（例如用 Python 生成 100 万个 'a' 加上恶意 Payload），会导致正则引擎崩溃并返回 `false`，从而绕过匹配。
  - 多行模式 `%0a` 绕过：如果没有加 `m` 或 `s` 修饰符，`^` 和 `$` 只匹配整个字符串的开头和结尾。传入 `\nPayload` 可能绕过首字母检测。

## 5. 无字母数字 WebShell (Non-Alphanumeric WebShell)

如果代码中存在 `eval()`，但通过正则 `/[a-zA-Z0-9]/` 过滤了所有字母和数字，如何执行命令？

- **异或 (XOR)**: `'^' ^ '?'` 得到 `'A'`。可以通过非字母数字字符异或构造出 `system` 和 `ls`。
- **取反 (Negation)**: `~("\x8f\x97\x8f\x96\x91\x99" ^ "\xff\xff\xff\xff\xff\xff")` 构造函数名。
- **自增 (Increment)**: 字符自增特性 `'a'++` 变成 `'b'`。
- *对策*：遇到这种题目，直接使用 Python 编写脚本，在本地自动生成所需的异或或取反 Payload 再发送。
