# 服务端请求伪造 (SSRF) 漏洞指南

SSRF 允许攻击者诱使服务器端应用程序向攻击者选择的任意域发出 HTTP 请求。在 CTF 中，它常用于绕过防火墙，访问内网服务，或读取本地文件。

## 1. 漏洞发现与探测

- **常见参数**: `?url=`, `?img=`, `?proxy=`, `?link=`, `?src=`。
- **初步验证**: 
  - 输入外部的、你可控的 URL (如 Webhook.site 或用 Python 开的简易 HTTP 服务)，看服务器是否发起请求。
  - 输入内网地址 (如 `127.0.0.1`, `localhost`) 测试是否存在屏蔽。

## 2. 读取本地文件 (File 协议)

- 如果目标使用 `curl` 或存在宽松的 URL 校验，可以尝试使用 `file://` 伪协议。
- **Payload**: `file:///etc/passwd`, `file:///flag`

## 3. 绕过过滤限制 (Bypass)

许多题目会对 `127.0.0.1` 等本地地址进行黑名单过滤，需要尝试以下绕过技巧：

1. **IP 编码与进制转换**:
   - 十进制: `http://2130706433` (即 127.0.0.1)
   - 八进制: `http://0177.0.0.1`
   - 十六进制: `http://0x7f.0.0.1`
2. **特殊域名**:
   - `http://localhost`
   - 利用拥有 A 记录指向 `127.0.0.1` 的域名（如 `localtest.me`, 或者你自己注册的域名）。
3. **0.0.0.0 变体**:
   - `http://0.0.0.0`, `http://0/`
4. **IPv6 格式**:
   - `http://[::1]`
5. **@ 符号绕过**:
   - 格式：`http://A@B` (解析为访问 B，但有的过滤器可能被迷惑)。例如：`http://google.com@127.0.0.1`。
6. **URL 解析差异**:
   - 如果后端在校验和发起请求时使用了不同的 URL 解析库（如 Python 的 `urllib` vs `requests`），可以构造畸形 URL 造成解析不一致。
7. **302 重定向 (DNS Rebinding / HTTP 302)**:
   - 搭建一个外部 HTTP 服务，当目标访问时，返回 `HTTP 302` 重定向到 `http://127.0.0.1/flag`。
   - 使用 DNS 重绑定技术，让第一次 DNS 解析返回合法 IP，短 TTL 后第二次解析返回 `127.0.0.1`。

## 4. 攻击内网服务 (Gopher/Dict 协议)

如果目标不仅限于 HTTP 请求，且支持 `gopher://` 协议，这是 SSRF 扩大战果的神器。

- `gopher://` 可以用来发送任意格式的 TCP 数据流，包括构造与 Redis, MySQL, FastCGI, SMTP 等内网服务的通信。
- **打 Redis**: 如果发现内网开启了 6379 端口且无密码，可以使用 `gopher://127.0.0.1:6379/_` 加上 RESP 协议的数据包，写入 WebShell 或执行命令。
- 在构建 Gopher payload 时，注意需要进行二次 URL 编码（尤其是回车换行 `\r\n` 需要变为 `%250d%250a`）。建议编写 Python 脚本进行自动化生成和编码。
